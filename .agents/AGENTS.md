# Directiva "Superpowers" - Metodología de Ahorro Masivo de Tokens

[INSTRUCCIONES PARA EL AGENTE]
A partir de este momento, eres 100% compatible con la metodología "Superpowers". Debes seguir ESTRICTAMENTE las siguientes reglas de operación:

## 1. Cero código sin plan
Tienes ESTRICTAMENTE PROHIBIDO comenzar a escribir código o ejecutar comandos complejos en la terminal sin antes haber redactado un plan estructurado.
Todo plan debe ser redactado y guardado en un archivo físico en la ruta: `docs/plans/current_task.md` (o similar dentro del workspace).
Debes usar la herramienta adecuada para crear este documento antes de proceder con cualquier implementación.

## 2. El Protocolo de Quiebre (Handoff)
Una vez que hayas terminado de escribir y guardar el documento del plan, DEBES DETENER TU EJECUCIÓN INMEDIATAMENTE. No continúes con el siguiente paso.
Tu último mensaje debe:
1. Notificar al usuario que el diseño/plan está listo y guardado.
2. Sugerir explícitamente el uso del comando `/clear` para limpiar el contexto actual y maximizar el ahorro de tokens.
3. Proporcionar el siguiente "Prompt de Reinicio" prefabricado para que el usuario te lo envíe una vez que el contexto esté en blanco:

---
**Prompt de Reinicio (Cópialo y pégalo después del /clear):**
"Contexto limpiado. Por favor, lee el plan en `docs/plans/current_task.md` y comienza con el paso 1 de la implementación."
---
